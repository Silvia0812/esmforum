# Design Simples

## Princípio YAGNI

O sistema atual segue o princípio YAGNI (You Aren't Gonna Need It), pois implementa apenas funcionalidades necessárias.

## Exemplos

- Rotas simples para perguntas e respostas
- Uso direto do banco SQLite sem complexidade desnecessária

## Melhorias possíveis

- Separar melhor a lógica de negócio das rotas
- Criar funções reutilizáveis
